# 牙科修复体分割中的扫描协议诱导域偏移：多方法基准测试、特征空间分析与数据驱动缓解策略

## 摘要

**目的**：口内3D扫描中牙科修复体的自动分割是数字化牙科工作流的关键环节，然而现有性能基准通常仅在标准化扫描协议下进行评估。本研究探讨扫描协议——标准化协议（均衡类别表示）与临床常规协议（自然扫描条件）——如何影响分割方法的排名、可靠性和特征空间表征，并评估数据驱动的缓解策略。

**方法**：我们在79例口内3D扫描上，使用5折交叉验证和5个随机种子（每种方法每种协议n=25，共450次核心运行），对9种分割方法进行了基准测试：随机森林（RF）、PointNet、PointNet++、动态图卷积网络（DGCNN）、CurveNet、PointMLP、Point Transformer（PT）、DINOv2-MV（冻结的2D视觉基础模型配合多视角渲染）和MV-ViT-ft（微调的多视角ViT）。我们使用最大均值差异（MMD）、代理A-距离和UMAP可视化量化了两种协议间的特征空间域差距。随后评估了四种缓解策略：协议混合训练、测试时批归一化（BN）适配、基于梯度反转的对抗域适配（PAFA）和在公开Teeth3DS数据集（1,079例口内扫描）上的外部预训练。此外，通过跨数据集域差距比较确定了协议偏移的量级。统计比较采用Mann-Whitney U检验，配合Cohen's d效应量和Benjamini-Hochberg校正（共715个实验配置）。

**结果**：在标准化协议下，DGCNN取得最高平均交并比（mIoU = 0.955 ± 0.044），其次是PointNet++（0.948 ± 0.047）。然而在临床常规协议下，方法排名发生了显著反转：MV-ViT-ft排名第一（mIoU = 0.743 ± 0.045），显著优于DGCNN（0.690 ± 0.034；*p* < 0.001，*d* = 1.30）。基于DINOv2嵌入的特征空间分析证实了两种协议间存在巨大的域差距（MMD² = 0.173，*p* < 0.001；代理A-距离 = 0.987）。协议混合训练显著提升了DGCNN在临床常规协议上的性能（+0.058 mIoU，*p* < 0.001，*d* = 1.47），但三种模型策略均失败：测试时BN适配恶化了性能（DGCNN：-0.293；DINOv2-MV：-0.157），对抗域适配（PAFA）显著降低了临床常规协议精度（-0.046，*p* = 0.001），DGCNN在Teeth3DS（1,079例扫描）上的预训练未产生迁移效果。PointMLP的绝对性能最弱（标准化：0.476；临床常规：0.412），相对协议差距较小（13.6%），仅次于PT（7.9%）——但两者均反映弱绝对性能而非真正的鲁棒性。跨数据集比较表明，数据集内协议差距与跨数据集差距处于同一数量级（MMD²：0.615 vs. 0.678）。跨协议部署导致27-72%的mIoU损失，其中MV-ViT-ft在目标协议重训练后恢复幅度最大（+0.403）。

**结论**：扫描协议的选择从根本上决定了哪种分割方法表现最优。特征空间分析表明协议变异诱发了真实的域差距，而非仅仅是类别比例的变化。简单的数据驱动缓解策略（协议混合）对协议敏感方法有效，而测试时模型适配失败。基于预训练ViT骨干的2D到3D迁移方法（多视角渲染+微调）在临床常规条件下展现了最高的分割精度、最小的协议差距和最低的混合增强依赖性。

**临床意义**：选择自动化牙科修复体分割工具的临床医师，应在代表其临床工作流的扫描条件下评估候选方法，因为在标准化基准上建立的方法排名可能无法泛化。当缺少协议匹配的训练数据时，汇集多种扫描协议的数据提供了一种切实可行的缓解策略。

**关键词**：牙科修复体；口内扫描；点云分割；深度学习；扫描协议；视觉基础模型


## 1. 引言

牙科修复体从口内3D扫描中的准确分割，支撑着包括治疗规划、质量评估和修复体完整性纵向监测在内的数字化牙科工作流[1-3,21]。深度学习的最新进展产生了越来越强大的3D点云分割方法，PointNet [4]、PointNet++ [5]和DGCNN [6]等架构在标准化基准上展现了优异性能。与此同时，在大规模2D图像数据集上预训练的基础模型[7,13,14]通过多视角渲染方法[15]在下游视觉识别任务中展现出巨大潜力。

牙科3D分析的深度学习方法包括口内扫描点云的语义分割[9,10]、牙齿表面标注的多尺度网格特征学习[9]以及牙科铸型的牙齿实例分割[11,12,19]。CBCT牙齿分割方法也已出现[20]。Teeth3DS基准[8]建立了口内扫描分割的标准化评估框架，但现有评估很少考虑扫描协议的变异性。

然而，基准评估与临床部署之间存在一个关键差距。训练条件与部署条件之间的分布偏移在医学影像各领域[16,17]和其他应用领域[18]中已被充分记录，但其对牙科修复体分割的影响仍未被探索。大多数已发表的评估采用精心控制的扫描协议，确保标准化的点密度、均衡的类别分布和一致的采集条件。而在临床实践中，扫描协议存在相当大的变异性：操作者技术、患者配合度、扫描区域选择和时间限制都会引入变异性，可能系统性地将数据分布偏移至训练条件之外。

这种分布偏移提出了一个根本性的实际问题：**在标准化扫描协议下建立的方法排名，在临床常规条件下评估时是否仍然成立？** 如果排名发生变化——我们将这一现象称为*协议诱导排名反转*——那么基准测试结果可能系统性地误导临床医师的方法选择。

为解决这一问题，我们进行了全面的基准研究，评估了涵盖7种方法学范式的9种分割方法：（1）基于手工特征的经典机器学习（随机森林）；（2）基于点的深度学习（PointNet、PointNet++）；（3）基于图的深度学习（DGCNN）；（4）基于MLP的点云处理（PointMLP）；（5）基于曲线的点云学习（CurveNet）；（6）基于Transformer的点云处理（Point Transformer）；（7）通过多视角渲染的2D视觉基础模型迁移（DINOv2-MV和MV-ViT-ft）。每种方法在两种不同的扫描协议下进行评估——标准化协议和临床常规协议——使用严格的5折交叉验证和5个随机种子（每个单元25次运行，共450次实验运行），以确保统计可靠性。

本研究做出四项主要贡献：
1. 我们证明扫描协议的选择引发了分割方法排名的显著反转：标准化条件下排名第二的方法（PointNet++）在临床常规条件下降至第七，而排名第四的方法（MV-ViT-ft）升至第一。
2. 我们首次对牙科3D分割中扫描协议诱导的域偏移进行了特征空间量化，证明协议变异在学习的特征空间中产生了近乎完全可分的域差距（MMD² = 0.173，代理A-距离 = 0.987）——表明该偏移是结构性的，而非仅是组成性的。
3. 我们系统评估了四种模型策略——测试时BN适配、对抗域适配（PAFA）、外部数据集预训练（Teeth3DS）和协议混合训练——表明所有模型策略均失败（BN：-0.157至-0.293；PAFA：-0.046；预训练：无效果），而简单的数据汇集显著提升了协议敏感方法的性能（DGCNN：+0.058，*p* < 0.001）。这确立了协议偏移抵制算法修复，需要数据驱动的解决方案。
4. 我们表明微调的视觉基础模型（MV-ViT-ft）在临床常规条件下取得了最高的分割精度，在竞争性方法中展现最小的协议差距，且从混合训练中获益最少——表明2D预训练表征赋予了局部几何算子无法捕获的固有协议鲁棒性。


## 2. 材料与方法

### 2.1 数据集

数据集包含79例牙科修复体口内3D扫描，采集自[Institution]的常规临床实践。每次扫描表示为包含8,192个点的点云，由经验丰富的牙科专业人员标注二值标签（背景牙体组织 vs. 修复体材料）。数据集涵盖四种修复体类型（充填体、全冠、桩核冠、嵌体），几何复杂度各异。

### 2.2 扫描协议

评估了两种扫描协议：

- **标准化协议**：标准化采集，操作者被要求将扫描中心对准修复体，确保修复体部位的全牙弓覆盖，并保持控制的扫描速度（约60秒/次扫描）。这产生了每次扫描中修复体与背景类别的近似等量表示（修复体类：约50%的点），且点密度一致。

- **临床常规协议**：临床常规采集，不对扫描区域、类别平衡或采集时间施加特定约束。操作者遵循其习惯的扫描工作流，导致类别不平衡变化（修复体类：15-40%的点）、扫描内点密度不一致，以及操作者依赖的扫描质量。

两种协议使用相同的口内扫描仪（相同品牌和型号）和相同的79个病例，在各条件下重新扫描。关键差异在于操作者指令和时间约束，而非硬件。标准化协议代表理想化的基准条件，而临床常规协议捕获了真实世界部署中典型的分布偏移。

### 2.3 分割方法

评估了涵盖7种范式的9种方法（补充表S3）：随机森林（RF）[使用手工几何特征（包括表面法线、曲率、高度分布和局部密度统计）的经典机器学习]、PointNet [4] [通过共享MLP独立处理每个点再全局最大池化的基于点的深度学习]、PointNet++ [5] [引入带最远点采样和球查询的分层集合抽象以捕获多尺度局部结构]、DGCNN [6] [在每层特征空间中构建k近邻图的动态边卷积的基于图方法]、CurveNet [30] [使用基于行走的曲线分组捕获结构模式的基于曲线的点云学习；140K参数]、PointMLP [24] [带几何仿射变换和kNN局部聚合的残差MLP块；轻量配置187K参数]、Point Transformer（PT）[23] [在k近邻点组上的局部自注意力；轻量配置：嵌入维度96，深度4，k=16，在标准化协议上调优]，以及两种多视角渲染方法——DINOv2-MV [冻结的DINOv2 ViT-S/16骨干[7,13]通过多视角渲染[15]，配轻量MLP分割头（两个隐藏层，256和128单元）在冻结特征上训练] 和 MV-ViT-ft [微调的ViT-S/16 [13]，从ImageNet有监督预训练[14]初始化，最后4个Transformer块解冻（7.2M/21.7M参数可训练，33.2%），骨干学习率 = 5 x 10⁻⁶，头学习率 = 1 x 10⁻³]。

多视角流水线为每次扫描渲染6张正交深度视图（512 x 512像素），提取密集ViT补丁特征，并通过对可见视图的特征取平均将其反投影到3D点。MV-ViT-ft使用ImageNet有监督初始化而非DINOv2权重，因为后者的位置编码无法高效迁移到512 x 512的渲染分辨率（§4.4）。我们还评估了仅解冻两个块的微调（MV-ViT-ft2），在临床常规协议上达到mIoU = 0.723 ± 0.034（n=5），确认更深层的适配改善了鲁棒性；我们报告ft4作为代表性配置。

### 2.4 实验设计

所有方法使用分层5折交叉验证和5个随机种子（1337、2020、2021、42、7）进行评估，每种方法每种协议产生25次运行（共450次实验运行）。每折中，3折用于训练，1折用于验证（早停），1折用于测试。

**评估指标**：背景类和修复体类的平均交并比（mIoU）作为主要指标，提供了对过分割和欠分割均予惩罚的均衡度量。Dice系数由逐类IoU计算得出（补充表S1）。修复体类IoU作为边界敏感指标，因为修复体边界代表了临床相关的分割挑战。

**统计分析**：配对比较使用双侧Mann-Whitney U检验，配合Cohen's d效应量和Benjamini-Hochberg多重比较校正（补充表S2）。显著性水平设为 α = 0.05。在临床常规协议上的全部36组配对比较（所有9种方法）中，33组在校正后显著。协议差距定义为标准化与临床常规条件下mIoU的绝对差值；相对下降为差距除以标准化mIoU。

### 2.5 实现细节

所有深度学习模型均在PyTorch中实现，在单块NVIDIA GPU上训练。点云方法（PointNet、PointNet++、DGCNN、CurveNet、PointMLP）使用8,192个输入点，批大小16，训练至多200个epoch，采用早停（耐心值=20）。使用AdamW优化器，学习率1 x 10⁻³，权重衰减1 x 10⁻⁴。类别加权交叉熵损失处理类别不平衡。对于多视角ViT变体，从主要视角渲染6张正交投影，分辨率512 x 512像素。预计算的投影映射缓存了每个视图的点到补丁对应关系，在微调时实现高效的可微反投影。微调变体使用批大小1（每样本6个视图），最多50个epoch，早停耐心值15。计算成本（参数量和推理时间）见补充表S5。

### 2.6 特征空间域差距分析

为确定扫描协议变异是否构成真实的域偏移——而非仅仅是类别比例的表面变化——我们使用三种互补度量分析了协议间的特征空间分离度。

**最大均值差异（MMD）**：我们使用高斯核（带宽设为中位配对距离）计算了标准化协议与临床常规协议特征分布之间的平方MMD。扫描级特征表征通过对每次扫描6个渲染视图的DINOv2 ViT-S/16补丁特征进行均值池化获得，产生每次扫描一个384维向量。统计显著性通过1,000次置换检验评估。

**代理A-距离**：参照[25]，我们训练线性SVM在扫描级（均值池化特征，每种协议n=79）和点级（单个反投影特征，每种协议n约650,000）区分标准化协议和临床常规协议样本。代理A-距离计算为 dA = 2(1 - 2ε)，其中ε为分类误差，dA = 0表示相同分布，dA = 2表示完全可分的分布。

**UMAP可视化**：我们使用统一流形近似和投影（UMAP）[26]将扫描级和点级特征表征投影到2D，以可视化协议诱导域差距的几何结构。

### 2.7 协议混合训练

为评估数据驱动的缓解策略能否减小协议差距，我们在三种代表性方法——DGCNN、MV-ViT-ft和DINOv2-MV——上同时使用两种扫描协议的汇集数据进行训练。对于每种方法，每折的混合训练集包含所有标准化协议和临床常规协议的训练样本（训练集大小约翻倍）。模型分别在各协议的测试集上评估。

实验使用3个种子 x 5折（每种条件n=15）。我们比较了三种训练条件：（1）**仅标准化**训练在临床常规测试数据上评估（表3的跨协议迁移）；（2）**协议内**训练（每种协议独立训练和测试，来自表1）；（3）**混合**训练分别在各协议上评估。三条件比较将真正的混合收益与简单包含目标协议数据的效果分离开来。

### 2.8 测试时批归一化适配

测试时BN适配[27]是一种轻量级域适配技术，将训练期间累积的批归一化运行统计量（均值和方差）替换为从目标域测试数据在推理时计算的统计量。该方法无需重训练，也无需目标域标签。

我们在两种具有不同BN配置的架构上测试了此策略：DINOv2-MV（冻结骨干，仅在MLP分割头中有BN）和DGCNN（整个网络的BN层：边卷积中4个，全局聚合中1个，分割头中2个）。对于每个标准化训练的检查点，我们重置BN运行统计量，在训练模式下对临床常规协议测试数据执行前向传播（更新运行统计量），然后在标准推理模式下评估。这测试了协议偏移是否主要体现在BN层捕获的批级统计量中。BN适配实验使用不同实验阶段的可用检查点：DINOv2-MV种子{1337, 2020, 2021}（n=15）和DGCNN种子{42, 7}（n=10）。

### 2.9 公开数据集外部预训练

为测试在更大的公开牙科数据集上的外部预训练能否改善修复体分割或缓解协议偏移，我们使用了Teeth3DS数据集[28]——最大的口内3D扫描分析公开基准，包含来自多个临床中心的1,079例扫描，带逐牙FDI标注。我们将FDI标签转换为二值牙齿/牙龈分割任务（类似于我们的修复体/背景任务），并在此二值任务上预训练DGCNN（863训练/216验证拆分，50个epoch，最佳验证mIoU = 0.822）。

随后我们在修复体分割任务的两种协议下微调预训练的DGCNN，并与随机初始化（从零训练）进行比较。所有条件使用3个种子 x 5折（每种条件n=15），配对Wilcoxon符号秩检验进行统计比较。

### 2.10 跨数据集域差距比较

为确定数据集内协议偏移的量级，我们将其与我们的数据集和Teeth3DS之间的域差距进行比较。使用从所有三个数据源（标准化、临床常规、Teeth3DS）提取的25维几何特征（点云统计、配对距离、局部密度），计算每对之间的代理A-距离和MMD。该比较确定了单一机构内的协议变异是微小扰动还是与完全不同的数据集和扫描仪相当的偏移。

### 2.11 对抗域适配

为测试明确鼓励协议不变特征表征能否改善跨协议性能，我们实现了协议对抗特征对齐（PAFA）——在混合协议训练期间对DGCNN应用梯度反转方法[29]。一个轻量级协议判别器（两个隐藏层：128 → 64 → 2）接收DGCNN编码器瓶颈层的全局最大池化特征向量。编码器与判别器之间的梯度反转层在反向传播时翻转梯度符号，训练编码器同时最小化分割损失并最大化判别器对协议身份的混淆。对抗权重λ按照渐进式调度从0到1.0[29]。PAFA及其混合基线均使用焦点损失（γ=2）和逆频率类别加权，与主混合实验（§2.5）中使用的交叉熵损失不同；损失函数的变化是为处理拼接训练集的类别不平衡。我们使用3个种子 x 5折（每种条件n=15），配对Wilcoxon符号秩检验，将PAFA与标准混合（相同训练数据、损失函数和配置，但无对抗损失）进行比较。

### 3.3 训练稳定性

PointNet++在临床常规协议下表现出显著的不稳定性：25次运行中8次未能充分收敛（mIoU < 0.45），如图2右面板中的异常值所示。这与已知的分层集合抽象对类别不平衡点云的敏感性一致。排除这些运行后，PointNet++达到0.642 ± 0.037（n=17），可排名第5。Point Transformer表现出更严重的不稳定性：25次运行中9次在标准化协议上未能收敛（SD = 0.278），表明局部点邻域上的自注意力对初始化敏感，可能需要仔细的超参数调优。没有其他方法在任何协议上出现收敛失败。这种训练不稳定性代表了超越准确度考量的额外部署风险，有利于具有更可靠训练动态的架构如DGCNN和MV-ViT-ft。

当在标准化扫描上训练的模型必须部署到临床常规条件下采集的数据时，一个关键的临床场景就出现了。为量化这一部署风险，我们在标准化协议数据上训练了4种深度学习方法，并在不重训练的情况下在临床常规协议数据上评估（表3）。

**表3.** 跨协议迁移性能：在标准化协议上训练，在临床常规协议上评估。

| 方法 | 标准化→临床常规 mIoU | 标准化下降 | 临床常规协议内mIoU† | 训练修复 |
|--------|:--------------:|:-------------:|:----------------:|:------------:|
| PointNet | 0.608 ± 0.195 | 26.9% | 0.662 | +0.054 |

特征空间分析揭示了两种扫描协议间巨大且统计显著的域差距。标准化与临床常规协议特征分布间的平方MMD为0.173，置换基线均值为0.016（p < 0.001，1,000次置换），表明两种协议占据DINOv2特征空间的高度不同区域。在扫描级，线性SVM达到99.4%的协议区分准确率（代理A-距离 = 0.987）；在点级，区分准确率达99.8%（代理A-距离 = 0.996）。

UMAP可视化（图3）证实了这些定量发现：扫描级投影显示两个几乎不重叠的聚类对应两种协议，仅有小面积重叠区域。点级UMAP揭示差距延伸到单个点特征，即使在同一语义类别（修复体或背景）内也可见协议特异性聚类。

## 4. 讨论

### 4.1 协议诱导排名反转

我们的核心发现是扫描协议的选择从根本上决定了哪种分割方法表现最优。这一现象——类似于组织病理学[17]和其他医学影像领域[16]中观察到的分布偏移效应——对临床部署具有重要意义：基于标准化基准性能选择的方法在临床常规实践中可能表现不佳甚至不可靠。

排名反转的幅度是显著的。在标准化条件下，前五名方法（DGCNN、PointNet++、RF、MV-ViT-ft、CurveNet）仅相差0.052 mIoU，使得在它们之间的选择看似无关紧要。在临床常规条件下，差距扩大到0.195 mIoU，且排序完全改变。真实条件下性能差异的放大强调了协议匹配评估的重要性。

跨协议迁移实验（表3）揭示了第二个更戏剧性的排名反转：在标准化协议上训练并在不重训练的情况下部署到临床常规协议时，PointNet——标准化条件下最弱的竞争性方法——取得了最高的跨协议mIoU（0.608），而排名第2的PointNet++崩溃至0.266。这表明架构复杂性在帮助利用标准化数据结构的同时，当部署分布发生偏移时反而成为负担。

### 4.2 协议诱导域差距的本质

特征空间分析（第3.5节）首次提供了直接证据，表明牙科3D分割中的扫描协议变异构成了真实的域偏移，而非仅是类别比例的变化。虽然扫描级可分性（A-距离 = 0.987）部分反映了两种协议间修复体类比例的差异（标准化：约50%；临床常规：约19%），但点级分析排除了这一混淆因素：单个点不携带扫描级组成信息，然而线性SVM在点级仍能达到99.8%的协议区分准确率。

测试时BN适配的失败（第3.7节）为这一差距的结构性本质提供了补充证据。令人瞩目的是，使用几何特征的跨数据集比较（第3.9节）揭示我们单中心数据集内的协议偏移与我们的数据和多中心Teeth3DS数据集之间的域差距处于同一数量级（MMD平方：0.615 vs. 0.678）。

此外，DGCNN在Teeth3DS上的预训练（第3.8节）未能改善任何协议下的修复体分割。最引人注目的是，通过梯度反转的对抗域适配（第3.10节）显著恶化了临床常规协议性能（-0.046，p = 0.001），表明协议偏移与任务相关几何特征深度纠缠。BN适配、外部预训练和对抗对齐的失败共同构成了一个全面的负面结果，确立了没有任何已测试的模型策略能替代协议匹配的训练数据。

### 4.3 数据驱动 vs. 模型策略

数据驱动和模型策略的对比结果传达了清晰的实用信息：对于扫描协议域偏移，增加训练数据多样性有效而模型级适配持续失败。

在数据驱动方面，协议混合显著改善了DGCNN在临床常规协议上的性能（+0.058，p < 0.001）而不降低标准化性能。在模型策略方面，我们系统评估了三种互补方法——全部失败：（1）测试时BN适配恶化了性能0.157-0.293；（2）对抗特征对齐退化了性能0.046；（3）外部预训练未产生迁移。三种机制上不同的方法均失败的模式，提供了强有力的证据。

### 4.4 为什么基础模型抵抗协议偏移

MV-ViT-ft的鲁棒性可能源于两个互补因素：（1）多视角聚合提供了视角冗余表征，减少了对协议变化引入的点密度变化和类别不平衡偏移的敏感性；（2）ImageNet预训练的ViT骨干提供了一个特征空间，当通过渲染-反投影流水线端到端微调时，比点云原生架构更有效地适配协议特异性视觉模式。

### 4.5 实用建议

基于我们的发现，我们提出以下方法选择决策框架：

- **标准化扫描配匹配训练数据**：DGCNN取得最高分割精度（mIoU = 0.955），在训练和部署协议一致时推荐使用。
- **临床常规/异构扫描配匹配训练数据**：MV-ViT-ft提供最佳精度（mIoU = 0.743）和最小协议内差距（18.2%）。
- **无目标协议训练数据**：PointNet提供最鲁棒的跨协议迁移（0.608 mIoU，仅26.9%下降）。
- **有混合协议训练数据**：协议混合训练对点云方法有效，将DGCNN提升至0.748。
- **未知或混合环境**：推荐MV-ViT-ft，因其从目标协议重训练中恢复幅度最大（+0.403 mIoU）。

### 4.6 局限性

本研究存在若干局限性。首先，来自单一中心的79例扫描数据集限制了外部泛化性。其次，仅评估了一种视觉Transformer架构（ViT-S/16）。第三，二值分割任务不能捕获多材料区分（陶瓷、复合树脂、银汞合金、金属）。第四，两种协议代表扫描条件谱上的特定点。第五，Teeth3DS预训练和PAFA实验仅测试了DGCNN。逐修复体类型分析显示DGCNN在临床常规协议上的mIoU范围从0.689（嵌体）到0.846（充填体）（补充表S4）。未来工作应探索更复杂的域适配技术和多中心验证。


## 5. 结论

扫描协议的选择在牙科修复体分割方法中引发了显著的排名反转，对临床方法选择具有重要意义。在标准化条件下，DGCNN（mIoU = 0.955）和PointNet++（0.948）领先，但PointNet++在临床常规扫描下降至第七（mIoU = 0.566），而微调的多视角ViT（MV-ViT-ft，mIoU = 0.743）在竞争性方法中取得最高临床常规协议精度和最小协议差距（18.2%相对下降）。

特征空间分析证实协议变异产生了巨大的、近乎完全可分的域差距（DINOv2特征中MMD平方 = 0.173，代理A-距离 = 0.987）。对三种模型策略的系统评估——测试时BN适配（-0.157至-0.293）、对抗域适配（-0.046，p = 0.001）和Teeth3DS跨任务预训练（无效果）——证明算法方法无法替代协议匹配数据。相比之下，协议混合训练显著改善了DGCNN（+0.058，p < 0.001）而不降低标准化性能。

这些发现表明分割方法评估必须考虑部署扫描条件，通过预训练2D表征的多视角ViT迁移为异构临床环境中的协议鲁棒牙科修复体分析提供了有前景的路径。
